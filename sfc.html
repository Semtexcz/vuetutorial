Průvodce instalací

Struktura Vue projektu

  • složka public

    • obsahuje logo, které se zobrazuje v title webové stránky a obsahuje výchozí html file: index.html

  • složka assets

    • obsahuje soubory, které webová stránka obsahuje. Neujčastěji to jsou obrázky, css soubory, písma atd.

  • složka components

    • obsahuje Single file component, což jsou malé části webu jako tlačítka, navigační lišty, tabulky a spoustu dalších elementárních částí, z kterých se skládá web

  • složka router se stará o url adresu, více si řekneme později

  • složka views obsahuje také Single file component, ale stará se o celý layout ( rozvržení stránky)

  • soubor App.vue je vstupní komponenta celé aplikace

  • soubor main.js je vstupní soubor pro JS, kde je inicializoavaná aplikace

  • soubor package.json je konfiugurační soubor projektu a obecně všech JS knihoven. Obsahuje důležité informace o projektu a také knihovny, na kterých projekt závisí (dependencies) nebo na kterých záleží vývoj (devDependecies)

Single file component (SFC)

Koncept, který nám umožňuje rozdělit komponenty do jednotlivých souborů s *.vue příponou. Soubor obsahuje tři tagy, které oddělují logiku, styl a HTML:

  1. <script>
  2. <template>
  3. <style>

syntaxe:

    <template>
    <div class="example">{{ msg }}</div>
    </template>

<script>
export default {
  data() {
    return {
      msg: 'Hello world!'
    }
  }
}
</script>

<style>
.example {
  color: red;
}
</style>

<custom1>
  This could be e.g. documentation for the component.
</custom1>

Základní nastavení

  • HTML

    <script src="https://unpkg.com/vue@3"></script>
<script src="https://unpkg.com/vue-router@4"></script>

<div id="app">
  <h1>Hello App!</h1>
  <p>
    <!-- use the router-link component for navigation. -->
    <!-- specify the link by passing the `to` prop. -->
    <!-- `<router-link>` will render an `<a>` tag with the correct `href` attribute -->
    <router-link to="/">Go to Home</router-link>
    <router-link to="/about">Go to About</router-link>
  </p>
  <!-- route outlet -->
  <!-- component matched by the route will render here -->
  <router-view></router-view>
</div>
    • <router-view> je to tag ve kterém se zobrazuje obsah podle aktuální URL a promíta se tam komponenta, kterou chceme zobrazit

  • <router-link> je to obyčejný a href odkaz, který po kliknutí změní URL a v zobrazí definovanou komponentu

  • JS 

  // 1. Define route components.
// These can be imported from other files
const Home = { template: '<div>Home</div>' }
const About = { template: '<div>About</div>' }

// 2. Define some routes
// Each route should map to a component.
// We'll talk about nested routes later.
const routes = [
  { path: '/', component: Home },
  { path: '/about', component: About },
]

// 3. Create the router instance and pass the `routes` option
// You can pass in additional options here, but let's
// keep it simple for now.
const router = VueRouter.createRouter({
  // 4. Provide the history implementation to use. We are using the hash history for simplicity here.
  history: VueRouter.createWebHashHistory(),
  routes, // short for `routes: routes`
})

// 5. Create and mount the root instance.
const app = Vue.createApp({})
// Make sure to _use_ the router instance to make the
// whole app router-aware.
app.use(router)

app.mount('#app')

// Now the app has started!

Single file component

//../src/router/index.js
import { createRouter, createWebHistory } from 'vue-router'
import Home from '../views/Home.vue'

const routes = [
  {
    path: '/',
    name: 'Home',
    component: Home
  },
  {
    path: '/about',
    name: 'About',
    // route level code-splitting
    // this generates a separate chunk (about.[hash].js) for this route
    // which is lazy-loaded when the route is visited.
    component: () => import(/* webpackChunkName: "about" */ '../views/About.vue')
  }
]

const router = createRouter({
  history: createWebHistory(process.env.BASE_URL),
  routes
})

export default router
//../src/main.js
import { createApp } from 'vue'
import App from './App.vue'
import router from './router'

createApp(App).use(router).mount('#app')

Po inicializaci dostaneme přístup k metodám this.$route a this.$router pro Composition API to jsou metody useRoute() a useRouter()

Dynamické routování

Je to routování, kde se nějaká část mění a je tedy proměnou jako například číslo uživatele /users/<id_uživatele> Syntaxe:

pro proměnnou používáme syntaxi s dvojtečkou, takže pro uživatele by to bylo takto: /users/:id 

const User = {
  template: '<div>User</div>',
}

// these are passed to `createRouter`
const routes = [
  // dynamic segments start with a colon
  { path: '/users/:id', component: User },
]

V komponentě pak to id získám přes $route.params.id v Composition API by to bylo useRoute().params.id 

const User = {
  template: '<div>User {{ $route.params.id }}</div>',
}

Matching syntax

Většina přístupů si vystačí se statickým routováním /about a s dynamickým, ale Vue Router umí daleko více přístupů. Když použijeme dynamické routování, tak místo výrazu :userId se vnitřně použije regexp ([^/]+), což najde alespoň jeden znak, který není lomeno.

Custom Regexp

Může se stát, že na stejné routě budeme chtít rozeznávat dvě věci jako například: /:orderId a /:productName. Pro tyto účely můžeme napsat vlastní regexp do závorky takto:

const routes = [
  // /:orderId -> matches only numbers
  { path: '/:orderId(\\d+)' },
  // /:productName -> matches anything else
  { path: '/:productName' },
]

Dvě lomítka jsou u \\d+ jako escape charakter

Opakované parametry: Někdy se může stát, že máme velmi zanořenou strukturu /one a /one/two atd. V takovém případě to můžeme vyřešit takto:

const routes = [
  // /:chapters -> matches /one, /one/two, /one/two/three, etc
  { path: '/:chapters+' },
  // /:chapters -> matches /, /one, /one/two, /one/two/three, etc
  { path: '/:chapters*' },
]

Nested routes

const routes = [
  {
    path: '/user/:id',
    component: User,
    children: [
      {
        // UserProfile will be rendered inside User's <router-view>
        // when /user/:id/profile is matched
        path: 'profile',
        component: UserProfile,
      },
      {
        // UserPosts will be rendered inside User's <router-view>
        // when /user/:id/posts is matched
        path: 'posts',
        component: UserPosts,
      },
    ],
  },
]

Komponenta User musí obsahovat tag <router-view>, kde se rendruje obsah children komponent

Navigace v JS kódu

Změnit URL adresu a přepnutí na jiné View/komponentu lze udělat i z JS kódu a nikoliv pomoci tagu <router-link>. Programově změním adresu pomocí metody this.$router.push():

// literal string path
router.push('/users/eduardo')

// object with path
router.push({ path: '/users/eduardo' })

// named route with params to let the router build the url
router.push({ name: 'user', params: { username: 'eduardo' } })

// with query, resulting in /register?plan=private
router.push({ path: '/register', query: { plan: 'private' } })

// with hash, resulting in /about#team
router.push({ path: '/about', hash: '#team' })

Možnosti jak poskládat cestu:

const username = 'eduardo'
// we can manually build the url but we will have to handle the encoding ourselves
router.push(`/user/${username}`) // -> /user/eduardo
// same as
router.push({ path: `/user/${username}` }) // -> /user/eduardo
// if possible use `name` and `params` to benefit from automatic URL encoding
router.push({ name: 'user', params: { username } }) // -> /user/eduardo
// `params` cannot be used alongside `path`
router.push({ path: '/user', params: { username } }) //

Navigace na novou URL adresu bez uložení historie (this.$router.replace()):

router.push({ path: '/home', replace: true })
// equivalent to
router.replace({ path: '/home' })

Tlačítko dopředu a zpět:

  • dopředu:

    this.$router.go(1)
//nebo
this.$router.forward()

  • dozadu:

    this.$router.go(-1)
//nebo
this.$router.backward()

Základní koncept

Store má tři koncepty state, getters a actions. Tyto koncepty odpovídají třem konceptům z vue: data, computed a methods

Definice storu

- ve složce `src` vytvoříme složku `stores`, kde budeme ukládat všechny state managment systémy
- ve složce vytvoříme soubor `store.js`, název může být libovolný
- ukázka souboru s definovaným storem:
import { defineStore } from 'pinia'

export const useStore = defineStore(
    'main', {
        state: () => {
            return {
                counter:0,
                name: 'Eduardo',
                isAdmin: true
            }
        }
    }
)

Použití storu

<template>
 <div class="store">
   <pre>{{ store }}</pre>
   <pre>{{ store.counter }}</pre>
   <pre>{{ store.name }}</pre>
 </div>
</template>

<script>
import {useStore} from '@/stores/store'

export default {
 name: 'HelloWorld',
 props: {
   msg: String
 },
 setup() {
   const store = useStore()

   return {
     store
   }
 }
}
</script>

Stav storu (CRUD) - state

<script>
import {useStore} from '@/stores/store'

export default {
  name: 'HelloWorld',
  props: {
    msg: String
  },
  setup() {
    const store = useStore()

    return {
      store
    }
  },
  methods: {
    editStore() {
      this.store.counter = 69
      console.log(this.store.counter)
    },
    resetStore() {
      this.store.$reset()
      console.log(this.store.counter)
    },
    mutateStore() {
      this.store.$patch({
        counter: 14,
        name: "Daniel"
      })
    },
    replaceStore(){
      this.store.$state = {
        counter:68,
        name: "Jaromír Jágr"
      }
    }
  }
}
</script>

export const useStore = defineStore(

Actions

  • ekvivalentní k Methods v komponentě

  • snaha je mít všechny funkce související se storem právě ve storu podle principu DRY. V jiném případě bychom v každé komponentě zbytečně opakovali ten samý kód.

  • příklad: store.js:

import { defineStore } from 'pinia'
import axios from "axios";

export const useStore = defineStore(
    'main', {
        state: () => {
            return {
                userData: null
            }
        },
        actions: {
            async getUsers() {
                try {
                    const response = await axios.get('https://reqres.in/api/users');
                    this.userData = response.data.data;
                } catch (error) {
                    console.error(error);
                }
            }
        }
    }
)

<komponenta>.vue :

<template>
  <div class="hello">
    <pre>{{ store }}</pre>
    <button @click="store.getUsers">Get users</button>
  </div>
</template>

  }
