Event- handling (Události)

Události jsou v programování nějaké situace, na které chceme reagovat. Obecně se tomuto stylu programování říká: [Event-driven programming](Event-driven programming - Wikipedia). Příklad takové události je kliknutí na něco, přejetí myši přes něco, scrollování, psaní na klávesnici atd.

[Syntaxe](Built-in Directives | Vue.js)

<button v-on:click="doThis"></button>

nebo zkráceně

<button @click="doThis"></button>

Zde jsme přidali na tlačítko událost po kliknutí. Když klikneme na tlačítko, tak se spustí funkce doThis.

Příklad

  • HTML

    <!DOCTYPE html>
<html lang="en">
<head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>Document</title>
</head>
<body>
    <div id="event-handling">
        <p>{{ message }}</p>
        <button v-on:click="reverseMessage">Reverse Message</button>
    </div>

    <script src="https://unpkg.com/vue@next"></script>
    <script src="04event_handling.js"></script>
</body>
</html>

  • JS

    const EventHandling = {
    data() {
      return {
        message: 'Hello Vue.js!'
      }
    },
    methods: {
      reverseMessage() {
        this.message = this.message
          .split('')
          .reverse()
          .join('')
      }
    }
  }

Vue.createApp(EventHandling).mount('#event-handling')

Cyklus directive v-for

Syntaxe:

očekává iterovatelný objekt: Array, Object, number, string, Iterable

 <div v-for="item in items">
 {{ item.text }}
 </div>

  • Lze iterovat i přes indexy, klíče a hodnoty

     <div v-for="(item, index) in items"></div>
 <div v-for="(value, key) in object"></div>
 <div v-for="(value, name, index) in object"></div>

  • Lze zvolit klíč, podle kterého lze řadit položky např. v objektu

     <div v-for="item in items" :key="item.id">
 {{ item.text }}
 </div>

Computed properties a Watchers

Computed properties jsou gettry, které vrací nějakou hodnotu jako jakýkoliv getter nějaké třídy. Byly zavedeny, aby v templatech nemusely být dlouhé výrazy jako např.: 

<!DOCTYPE html>
<html lang="en">
<head>
   <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>Document</title>
</head>
<body>
    <div id="computed-basics">
        <p>Has published books:</p>
        <span>{{ author.books.length > 0 ? 'Yes' : 'No' }}</span>
    </div>

    <script src="https://unpkg.com/vue@next"></script>
    <script src="12computedProps.js"></script>
</body>
</html>

Místo dlouhého <span>{{ author.books.length > 0 ? 'Yes' : 'No' }}</span> lze použít computed properties.

<!DOCTYPE html>
<html lang="en">
<head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>Document</title>
</head>
<body>
    <div id="computed-basics">
        <p>Has published books:</p>
        <span>{{ publishedBooksMessage }}</span>
    </div>

    <script src="https://unpkg.com/vue@next"></script>
    <script src="12computedProps.js"></script>
</body>
</html>
app = Vue.createApp({
    data() {
      return {
        author: {
          name: 'John Doe',
          books: [
            'Vue 2 - Advanced Guide',
            'Vue 3 - Basic Guide',
            'Vue 4 - The Mystery'
          ]
        }
      }
    },
    computed: {
      // a computed getter
      publishedBooksMessage() {
        // `this` points to the vm instance
        return this.author.books.length > 0 ? 'Yes' : 'No'
      }
    }
  })

vm = app.mount('#computed-basics')

    const App = {
    data() {
        return {
            question: '',
            answer: 'Questions usually contain a question mark. ;-)'
        }
    },
    watch: {
        // whenever question changes, this function will run
        question(newQuestion, oldQuestion) {
            if (newQuestion.indexOf('?') > -1) {
                this.getAnswer()
            }
        }
    },
    methods: {
        getAnswer() {
            this.answer = 'Thinking...'
            axios
            .get('https://yesno.wtf/api')
            .then(response => {
                this.answer = response.data.answer
            })
            .catch(error => {
                this.answer = 'Error! Could not reach the API. ' + error
            })
        }
    }
}

Na obrázku je zobrazen životní cyklus Vue aplikace. V každé fázi tohoto cyklu, lze zavolat nějaký skript. Červeným rámečkem je naznačena fáze, do které se dá vstoupit a přepsat běh programu. Těmto fázím se říká háky (hooks).

    const Counter = {
    data() {
      return {
        counter: 0
      }
    },
    beforeCreate() {
      console.log("beforeCreate")
      console.log('count is: ' + this.counter)
    },
    created() {
      var a = prompt("Zadej, cokoliv!")
      console.log('count is: ' + this.counter)

Vue Component

  • Pro vybudování velkých aplikací je Vue navrženo aby se dalo skládat z jednotlivých komponent (Vue Component ), což jsou malé části, které jsou znovupoužitelné.

  • Root a Instance komponenty

    • Každá Vue aplikace musí začít vytvoření její instancí Vue.createApp():
 const app = Vue.createApp({
  /* options */
})
  • instance aplikace je důležitá pro registraci Globální API, což jsou objekty, které jsou vyžadovány vždy celou aplikací a komponentami 
const app = Vue.createApp({})
app.component('SearchInput', SearchInputComponent)
app.directive('focus', FocusDirective)
app.use(LocalePlugin)
  • instanci můžeme zapisovat jako Chain:
Vue.createApp({})
  .component('SearchInput', SearchInputComponent)
  .directive('focus', FocusDirective)

Props

Props mohou být zadány jako list viz Syntaxe pro registraci komponenty je následující: nebo jako objekt, kde klíč je název property a hodnota je datový typ:

props: {
  title: String,
  likes: Number,
  isPublished: Boolean,
  commentIds: Array,
  author: Object,
  callback: Function,
  contactsPromise: Promise // or any other constructor
}

Předávat komponentě propertu můžeme staticky nebo dynamicky

  • staticky:

    <blog-post title="My journey with Vue"></blog-post>

  • dynamicky

    <!-- Dynamically assign the value of a variable -->
<blog-post :title="post.title"></blog-post>

<!-- Dynamically assign the value of a complex expression -->
<blog-post :title="post.title + ' by ' + post.author.name"></blog-post>

    Props přenáší data od rodičů dětem a v dětských komponentách by neměly být jakkoliv upravovány. Pokud chci propertu použít jako stav, tak bych si jí měl uložit do lokálních dat a nemanipulovat přímo s propertou.

    props: ['initialCounter'],
data() {
  return {
    counter: this.initialCounter
  }
}

    Pokud chci nějakým způsobem propertu přepočítat a vracet něco jiného, tak bych měl použít Computed Properties

    props: ['size'],
computed: {
  normalizedSize() {
    return this.size.trim().toLowerCase()
  }
}

Validace

Slouží k upozornění na jiný datový typ - Vue upozrňuje v konzoli Způsoby validace:

app.component('my-component', {
  props: {
    // Basic type check (`null` and `undefined` values will pass any type validation)
    propA: Number,
    // Multiple possible types
    propB: [String, Number],
    // Required string
    propC: {
      type: String,
      required: true
    },
    // Number with a default value
    propD: {
      type: Number,
      default: 100
    },
    // Object with a default value
    propE: {
      type: Object,
      // Object or array defaults must be returned from
      // a factory function
      default() {
        return { message: 'hello' }
      }
    },
    // Custom validator function
    propF: {
      validator(value) {
        // The value must match one of these strings
        return ['success', 'warning', 'danger'].includes(value)
      }
    },
    // Function with a default value
    propG: {
      type: Function,
      // Unlike object or array default, this is not a factory function - this is a function to serve as a default value
      default() {
        return 'Default function'
      }
    }
  }
})
