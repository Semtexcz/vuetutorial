Co je webpack?

Jedná se o tzv. static module bundler, což je program, který z JS modůlů běžících na Node.js vytváří statické soubory (.html, .js, .css atd.), které umí číst webové stránky. To nám umožňuje vyvíjet webové stránky lokálně a velký projekt "porcovat" na malé moduly.

Koncept

Web pack interně vytvoří graf závislostí a z použitých modulů vytvoří tzv. statické assety. To jak je vytvoří určuje konfigurační soubor.

Od verze 4.0.0 není konfigurační soubor zapotřebí

Entry point

Určuje, z kterých modulů má Webpack začít tvořit interní graf závislostí. Defaultní cesta je ./src/index.js, ale lze specifikovat v konfiguračním souboru

webpack.config.js

 module.exports = {
 entry: './path/to/my/entry/file.js',
 };

Output

Určuje kam má uložit zkompilované soubory a jak je má pojmenovat. webpack.config.js

const path = require('path');


module.exports = {
  entry: './path/to/my/entry/file.js',
  output: {
    path: path.resolve(__dirname, 'dist'),
    filename: 'my-first-webpack.bundle.js',
  },
};

Loaders

Webpack rozumí jen Javascriptu a JSON souborům. Loaders umožňují kompilovat i jiné typy souborů např. CSS.

Nastavit loader lze přes konfigurační soubor takto:

webpack.config.js

const path = require('path');

module.exports = {
  output: {
    filename: 'my-first-webpack.bundle.js',
  },
  module: {
    rules: [{ test: /\.txt$/, use: 'raw-loader' }],
  },
};

module obsahuje dvě klíčová slova test a use.

  • test nastavuje jaké soubory má Webpack uvažovat

  • use určuje jaký loader má být použitý

Plugins

Pluginy umožňují rozšířit funkčnost webpacku nad rámec loaderů.

konfigurace:

webpack.config.js

const HtmlWebpackPlugin = require('html-webpack-plugin');
const webpack = require('webpack'); //to access built-in plugins

module.exports = {
  module: {
    rules: [{ test: /\.txt$/, use: 'raw-loader' }],
  },
  plugins: [new HtmlWebpackPlugin({ template: './src/index.html' })],
};

Ve výše uvedeném příkladu html-webpack-plugin vygeneruje soubor HTML pro aplikaci a automaticky vloží všechny vygenerované balíčky do tohoto souboru.

Mode

Nastavuje režim vývoje (production, development a none), defaultně je nastaven mód production.

konfigurace:

module.exports = {
  mode: 'production',
};

Tutorial

Základní nastavení projektu

  1. vytvoření projektu

    mkdir webpack-demo
cd webpack-demo
npm init -y
npm install webpack webpack-cli --save-dev

  2. vytvoření souborové struktury

      webpack-demo
  |- package.json
  |- package-lock.json
  |- index.html
  |- /src
      |- index.js

  3. úprava souborů

src/index.js

  function component() {
  const element = document.createElement('div');

  // Lodash, currently included via a script, is required for this line to work
  element.innerHTML = _.join(['Hello', 'webpack'], ' ');

  return element;
  }

document.body.appendChild(component());

index.html

<!DOCTYPE html>

<html>
  <head>
    <meta charset="utf-8" />
    <title>Getting Started</title>
    <script src="https://unpkg.com/lodash@4.17.20"></script>
  </head>
  <body>
    <script src="./src/index.js"></script>
  </body>
</html>

  1. Vytvoření balíčku (bundle)

rozdělíme kód na distrubuci (./dist) a zdroj (./src) 

      webpack-demo
      |- package.json
      |- package-lock.json
+ |- /dist

+ |- index.html

  |- /src
  |- index.js

  1. Instalace potřebných knihoven V kódu používáme knihovnu loadash, a proto bychom jí měli přidat do projektu

    npm install --save lodash

  2. Importování knihovny lodash do src/index.js

  1. Odstraníme skripty z dist/index.html a přidáme jediný hlavní: main.js

  2. Nyní můžeme zavolat již Webpack a zbuildovat statickou webovou stránku

    Ještě před tím musíme změnit package.json, přidáním možnosti "build" :"webpack" do scripts:

    {
      "name": "webpack-demo",
      "version": "1.0.0",
      "description": "",
      "private": "true",
      "scripts": {
          "test": "echo \"Error: no test specified\" && exit 1",
          "build": "webpack"

      "keywords": [],
